    [ ] 11.6.4 Search tools — resolve the library by name down to playable

        [x] 11.6.4.1 search — {type: music|tv-show|movie, artist?, title?,
            season?, number?, limit?, offset?, count?} down to leaf-file
            rows {file, <id>, label}; descends to album/season only when
            named, so wide queries are one Kodi call; always reports total;
            substring title hitting several containers makes total a sum
            and offset best-effort
    [x] 11.6.5 playfile — play one file via Player.Open {item:{file}}
        (KODI-API 12.10.9); any path, in-library or not; auto-selects the
        audio/video player; Kodi enriches a scanned file back to its
        library item; returns the player_state() snapshot
    [x] 11.6.6 rpc — generic escape hatch: any method, returns Kodi's raw
        result verbatim (unshaped); gated per instance by allow_rpc in the
        config, granted only by hand-editing config.json — the model can
        never enable it itself
    [x] 11.6.7 queue — append (or insert-next at current+1) into the
        already-playing queue; caller starts playback first; playlistid < 0
        means nothing queueable (tool error); refuses a library id whose
        kind mismatches the active player (Kodi accepts it silently);
        remove action deferred; clear → dropplaylists, list → getplaylist
    [x] 11.6.8 getplaylist — read the current queue via Playlist.GetItems:
        rows {<type>id|file, label, type} + position of the now-playing
        item; explicit type arg reads an inactive playlist too; empty/none
        is an empty list, not an error (tests/getplaylist.test)
    [x] 11.6.9 dropplaylists — Playlist.Clear on all three playlists
        (audio 0 / video 1 / picture 2); clearing the active one leaves the
        current item playing, so it never interrupts playback
        (tests/dropplaylists.test)
